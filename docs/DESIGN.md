# 顔写真付き体制表 自動生成ツール 設計書

## 1. システム概要

クライアント（住宅事業集合建築本部）の人事異動時に、Excel体制表へ顔写真を自動配置するWindowsローカルアプリケーション。

## 2. 技術スタック

| レイヤ | 技術 | 理由 |
|---|---|---|
| 言語 | Python 3.10+ | openpyxl/PySide6 が成熟、配布実績豊富 |
| GUI | PySide6 (Qt6) | Windowsネイティブ風UI、商用利用可、テーブル/ファイル選択UIが標準装備 |
| Excel I/O | openpyxl | xlsx 読み書き＆画像挿入の標準ライブラリ |
| 画像処理 | Pillow | 写真サイズ調整・回転 |
| DB | SQLite (標準ライブラリ) | サーバー不要、単一ファイル、運用容易 |
| ORM | SQLAlchemy 2.x | 型安全・マイグレーション容易 |
| 配布 | PyInstaller | 単一.exeで配布可、Python未インストール環境でも動作 |

## 4. データモデル（DB）

### `employees` テーブル

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | 内部ID |
| name | TEXT NOT NULL | 漢字氏名（例: 藤堂　誠司） |
| name_kana | TEXT | カタカナ氏名（例: ﾄｳﾄﾞｳ ｾｲｼﾞ） |
| match_key | TEXT NOT NULL | テンプレート照合用キー（テンプレートに記載される短縮名: 例「藤堂」） |

| department | TEXT | 所属本部・部 |
| branch | TEXT | 支店 |
| section | TEXT | 課 |

| role_marks | TEXT | 記号（☆■◆○▽など）カンマ区切り |
| employment_type | TEXT | 雇用区分（正社員/契約/派遣/再雇用） |

| note | TEXT | 備考（産休等） |
| created_at | DATETIME | |
| updated_at | DATETIME | |

### `output_history` テーブル（任意）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | |
| template_path | TEXT | 使用テンプレート |
| output_path | TEXT | 出力ファイル |

### `mapping_overrides` テーブル（同名解決の学習）

| カラム | 型 | 説明 |
|---|---|---|
| id | INTEGER PK | |
| template_signature | TEXT | テンプレファイル名（または内容ハッシュ） |
| sheet_name | TEXT | シート名 |
| cell_address | TEXT | セル番地（例: M14） |
| match_key | TEXT | テンプレに書かれた文字列 |
| employee_id | INTEGER FK→employees.id | 紐付け先 |
| created_at | DATETIME | |

UNIQUE制約: (template_signature, sheet_name, cell_address)

### 同名解決ロジック

`match_key` でDB検索 → 候補数で分岐:
1. **0件**: 警告ログ「未登録」、スキップ
2. **1件**: 自動採用
3. **2件以上**:
   - まず `mapping_overrides` を参照 → ヒットすれば自動採用
   - 次に「シート名 → 所属/支店」のマッピングで絞込み（例: シート名「東京東」→ branch="東京東"）
   - それでも1件に絞れなければ、出力時に「曖昧マッチ解決ダイアログ」表示
   - 解決結果は `mapping_overrides` に保存（次回以降は自動）
4. **プレースホルダー詳細指定**: `{{photo:佐藤#id=42}}`, `{{photo:佐藤@東京南}}` で完全一意指定可能

## 5. テンプレート照合方式

**方式: プレースホルダー方式 + 氏名直接記載方式 のハイブリッド**

クライアントの既存テンプレートは「セルに氏名（短縮名）」が記載されているため、それを活かす。

### 写真配置ルール

テンプレートのセル走査時、以下のいずれかにマッチしたら、対応する写真を「指定オフセット」に挿入する。

1. **プレースホルダー方式（推奨・明示的）**
   - セル値が `{{photo:氏名}}` の場合 → そのセル位置に写真を貼り付け、セル値はクリア
   - 例: `C4` に `{{photo:藤堂　誠司}}` と書く → C4 にトドウの写真が貼られる
   - メリット: 位置が明示的で誤動作なし

2. **氏名検出方式（既存テンプレ互換）**
   - セル値が「人名らしい文字列」（DB の `match_key` または `name` と一致）の場合 → 設定で指定した相対オフセット（例: 同列・N行上）に写真を貼り付け
   - GUIで「テンプレ毎の写真オフセット設定」を持つ
   - 既存の手作業フローをそのまま自動化できる

両方式は同一テンプレ内で共存可能。プレースホルダーが優先。

### 写真サイズ・配置

- 写真のセル範囲（from/to）は固定の幅×高さ（例: 5列×6行）でデフォルト指定
- GUIで「写真サイズプリセット」を設定（テンプレ毎に保存）

### ステータスフィルタ

- `status = '休職中'` または `'退職'` の従業員はマッチ対象外（写真を貼らない）

## 9. エラーハンドリング方針

- **写真未登録**: スキップしてログに警告（処理続行）
- **氏名重複**: マッチした全員に同じ写真を貼り、重複警告ログ
- **テンプレートに該当氏名なし**: DB側の余剰従業員はログ通知
- **xlsxファイル破損/書き込み権限なし**: ダイアログでエラー表示

## 10. Windows配布

- `PyInstaller --onefile --windowed --icon=app.ico src/main.py` で単一.exe生成
- DB（employees.db）と photos/ フォルダはアプリと同階層に配置（書き込み可能な場所）
- 初回起動時にユーザーフォルダ配下（`%APPDATA%/Roster/`）にDBとphotosを自動展開する方式を採用
